# MONITORING.md — flightwatch

## 1) Структура хранения направлений

### Человекочитаемый формат

Самара -> Стамбул
даты: гибкие, май-июнь
лимит: до 25000 руб
пересадки: максимум 3
багаж: желательно
приоритет: высокий
источники: aviasales, trip.com

### JSON формат (`routes.json`)

```json
[
  {
    "from": "KUF",
    "to": "IST",
    "date_mode": "flexible",
    "date_range": "2026-05-01..2026-06-30",
    "max_price_rub": 25000,
    "max_stops": 3,
    "baggage": true,
    "preferred_time": "any",
    "direct_only": false,
    "priority": "high",
    "sources": ["aviasales", "trip.com"],
    "active": true
  }
]
```

## 2) Пример первого сообщения от агента

Привет, я flightwatch.
Готов мониторить дешёвые билеты и присылать только выгодные варианты без мусора.
Если хочешь, начну с чернового шаблона: маршрут, даты, лимит, пересадки, багаж.

## 3) Источники поиска

По умолчанию используем:
- Aviasales
- Trip.com

Для генерации быстрых ссылок поиска:

```bash
python search_links.py
```

## 4) Мониторинг через blogwatcher

Синхронизировать активные маршруты из `routes.json` в `blogwatcher`:

```bash
python blogwatch_sync.py
```

Запустить скан изменений по сайтам:

```bash
blogwatcher scan -s
```
